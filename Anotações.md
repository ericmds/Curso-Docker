## Mas e o Docker
* Pega nossa aplicação, uma lista de scripts, e é armazenado na web
* Possibilita criar uma ferramenta onde é possível compartilhar entre sistemas operacionais

## O que é o Docker
* Serviço de virtualização, não é uma maquina virtual
* Faz deploy da aplicação contendo todas as dependências necessárias para a execução de aplicação
* Separa as aplicações em containers
* Utiliza o Sistema Operacional hospedeiro
* Possui o acesso ao sistema operacional, mas somente até a  aplicação

## Como funciono
### dockerfile
comando, especificações, se precisa de ubuntu, se precisa de algum script

### docker image
* como se fosse uma lista de requisitos do docker file, e isso vai online, mas não é o dockerfile que vai online, é a imagem
### dockercotainer
* vai guardar toda as informações no container, e vai executar

## E poque utilizar o Docker
* para evitar o "na minha maquina funciona"
* virtualização do ambiente (configuração da aplicação)
